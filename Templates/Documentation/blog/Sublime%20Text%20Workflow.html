Single button to publish.

As I mentioned in the begining, this whole rabbit hole had its begining with the thought, I wish I could open up my text editor and write my blog post, and press a button to publish it. Some ideas are like cats, they demand attention. If we don't give it to them, they will intrude our personal space, our laptops, our thoughts. Hmm, finally I decided to give it a go. Turns out, themes for blogs could be created so easily, I made 5 out of the time I used to make a single TW5 theme. I even found a decent looking documentation web and snatch its looks too for this documentation.

Enough about that. Here I am shouwing how I finally got around to achieving my initial objective using a couple of plugins for sublime text 3.

The plugins I used are : Stringify and InsertDate. Install them both using Sublime text package installer.

Keybindings

Once you are done installing plugins, open Preferences > Key Bindings and add the following code to Right hand side plane.

KeyBindings -Sublime 
{ "keys": ["f3"], "command": "url_encode" },
{ "keys": ["f5"], "command": "insert_date", "args": {"format": "%Y%m%d%H%M%S001"} },

Snippet

Finally, go to Preferences > Browse Packages. Inside the User folder create a file with the extension .sublime-snippet for eg: tiddlerHeader.sublime-snippet. Add the following code and save.

tiddlerHeader.sublime-snippet 
<snippet>
<content><![CDATA[
title: ${TM_FILENAME/\.tid//g} 
tags: $2
created: $3
modified: $4
modifier: ${5:Riz}
summary: $6
caption: $7
type: ${8:blog/blog-post}

$0

]]></content>
<!– Optional: Set a tabTrigger to define how to trigger the snippet –>
<tabTrigger>tdh</tabTrigger>
<description>Tiddler header</description>
<!– Optional: Set a scope to limit where the snippet will trigger –>
<scope>text.tid</scope>
</snippet>

Now create a file with extension .tid in your tiddlers folder. On the first line, type tdh and press tab. If you followed the steps correctly, it would have expanded to

tiddlerHeader.sublime-snippet 
title: myFileName
tags: 
created:
modified:
modifier: Riz
summary:
caption:
type: blog/blog-post

A few points about the snippet for the uninitiated.

  • You can obviously change the modifier name in the original snippet to your name
  • It will jump to the next stop on pressing tab. Once you entered the tags, press tab and it will position itself after created. Once you reach created, press F5 key - it will enter the correct date-stamp format. Same for the modified field.
  • After the last field press tab to jump to the text field leaving one blank line.
  • This snippet will only work if your file has .tid extension.
  • Also, apart from the sinppet, the above steps also maps url encoding function to your F3 key. So if you want the double url encoded title for your href attribute, you just need to select the text and press F3 twice.

In the interest of full disclosure, I use AdvancedNewFile plugin for easy file creation, Emmet plugin for HTML work, and Shell Exec plugin for accessing the terminal occasionally.

Assuming that you are planning to host the blog in github or similar git powered targets, I recommend authenticating using SSH keys to avoid having to enter your password each time you push.
